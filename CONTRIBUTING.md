# Contributing to Discord Release Notifier

We welcome contributions to the Discord Release Notifier project!  Here's how you can help:

## Reporting Issues

If you find a bug or have a suggestion, please open an issue on GitHub. Be as specific as possible, including:

*   A clear and concise description of the issue.
*   Steps to reproduce the issue (if applicable).
*   Expected behavior vs. actual behavior.
*   Any relevant error messages or logs.

## Contributing Code

1.  **Fork the repository.**
2.  **Create a branch for your changes.** Use a descriptive name, like `fix-bug` or `add-feature`.
3.  **Make your changes.** Follow the project's coding style and conventions.
4.  **Test your changes.** If possible, add unit or integration tests to ensure your changes are working correctly.
5.  **Commit your changes.** Use clear and informative commit messages.
6.  **Push your branch to your forked repository.**
7.  **Create a pull request.**  Submit your changes for review.

## Code Style

Please follow the project's existing code style.  We use EditorConfig to help maintain consistency.

## Pull Request Guidelines

*   Keep pull requests focused and concise.
*   Address any feedback from reviewers.
*   Make sure your code is well-tested.
*   Update the documentation if your changes introduce new features or functionality.

## Getting Help

If you have questions or need help, feel free to ask in the GitHub Discussions or open an issue.
